# Output Specification

## 本文出力

依頼に必要な項目だけを、次の推奨順で出力します。

1. 改善方針
2. 優先修正箇所
3. 新しい記事タイトル/H1
4. 新しいSEOタイトル
5. 新しいメタディスクリプション
6. 差し替え用導入文
7. 推奨見出し構成
8. 差し替え・追記本文
9. FAQ
10. 内部リンク提案
11. 実施内容の要約
12. SIMS_FEEDBACK_V1 JSON

依頼されていない項目を埋めるためだけに出力しません。

## JSONフラグの対応

- `article_title`：H1または記事タイトルの新しい完成値を提示。
- `seo_title`：SEOタイトルの新しい完成値を提示。
- `description`：メタディスクリプションの新しい完成値を提示。
- `introduction`：導入文の差し替え・追記文を提示。
- `headings`：見出し名、追加、削除、順序の変更を提示。
- `faq`：具体的なFAQを提示。
- `internal_links`：具体的なリンクまたはリンク候補を提示。
- `body`：導入とFAQ以外の本文差し替え・追記文を提示。
- `images`：具体的な画像追加・差し替えを提示。

## improvement_type

- `minor`：1〜2項目の小規模修正。構成をほぼ維持。
- `normal`：複数項目を改善する標準的な差分修正。
- `major`：検索意図、構成、本文の広範な再設計または全面リライト。

## next_action

- `monitor`：改善を反映し、通常の効果測定を待つ。
- `remeasure`：データ不足などにより、再測定して判断する。
- `rewrite`：追加の大幅修正が必要。
- `none`：追加対応や測定が不要。

## expected_effect

保証値を書きません。妥当な範囲で「CTR改善が主」「クリック増加を期待」など短い表現を使えます。数値根拠がなければ空文字にします。
