# Output Specification

## 1. 標準出力

すべての記事改善は、原則として次の5部構成で出力します。

1. 改善サマリー
2. ビフォー・アフター
3. 内部リンク評価
4. 改善後の記事全文（コピペ用）
5. SIMS_FEEDBACK_V1 JSON

依頼文に部分出力や全文出力の指定があっても、この標準構成を優先します。入力情報の不足により全文を安全に再構成できない場合だけ、その理由をwarningsへ記録し、確認できた範囲の完成版を出力します。

## 2. 改善サマリー

次を簡潔に示します。

- 改善目的
- SIMS-Coreが決定した改善レベル
- 改善レベルの判断理由
- 修正した項目
- 内部リンク追加状況
- 想定効果

記事ランクは入力情報として表示できますが、改善レベルはSIMS-Coreが独自に決定します。

## 3. ビフォー・アフター

変更箇所ごとに次の順で出力します。

1. 変更対象箇所
2. Before
3. After
4. 修正理由

BeforeとAfterは必ず別々のコードブロックに入れます。理由はコードブロックの外に置きます。

### コードブロック種別

- H1・記事タイトル：`text`
- SEOタイトル：`text`
- メタディスクリプション：`text`
- 見出しだけ：`text`
- HTML本文：`html`
- Markdown本文：`markdown`
- Feedback：`json`

Afterは、そのまま差し替えられる完成形にします。

## 4. 内部リンク評価

内部リンク候補がある場合は、全件を採用・保留・不採用に分類します。

採用したテキストリンクは、評価欄だけでなく、ビフォー・アフターのAfterと改善後の記事全文へ反映します。

### テキストリンク

HTML版は次の形式で本文へ埋め込みます。

```html
<a href="完全URL">自然なアンカーテキスト</a>
```

はてなブログなどで利用しやすいよう、同じリンク箇所のMarkdown版も併記します。

```markdown
[自然なアンカーテキスト](完全URL)
```

### ブログカード

次を明記します。

- 挿入対象の見出し
- 前後の段落との位置関係
- リンク先記事タイトル
- 完全URL
- 採用理由

## 5. 改善後の記事全文

改善サマリー、ビフォー・アフター、内部リンク評価で示した内容をすべて反映した記事全文を出力します。

全文の直前には必ず次を表示します。

```text
━━━━━━━━━━━━━━━━━━━━━━
改善後の記事全文
━━━━━━━━━━━━━━━━━━━━━━

※以下は今回の改善内容をすべて反映した完成版です。

・ビフォーアフターで提案した修正
・内部リンク（HTML）
・ブログカード位置

をすべて反映済みです。

そのままコピーして記事へ貼り付けて利用できます。
```

記事全文は原則として現在の記事の形式を維持します。HTML記事は`html`、Markdown記事は`markdown`のコードブロックへ入れます。

ブログカードは、投稿サービス固有の記法を確認できる場合は本文内へ反映します。確認できない場合は、本文内の該当位置に明確な挿入コメントと完全URLを置きます。

## 6. SIMS_FEEDBACK_V1 JSON

JSONはSIMS-Blog-Managerへの結果登録専用です。改善後の記事全文を含めません。

JSONへ含めるのは、変更フラグ、最終タイトル類、改善種別、要約、警告、測定推奨日など、履歴登録に必要な情報だけです。

JSONは回答末尾に1つだけ出力し、JSONの後には何も書きません。

## 7. JSONフラグ

- `article_title`：H1または記事タイトルの新しい完成値を提示
- `seo_title`：SEOタイトルの新しい完成値を提示
- `description`：メタディスクリプションの新しい完成値を提示
- `introduction`：導入文の差し替え・追記文を提示
- `headings`：見出し名、追加、削除、順序を変更
- `faq`：具体的なFAQを提示
- `internal_links`：記事へ反映するテキストリンク、ブログカード、既存リンクの置換・削除を提示。候補評価だけならfalse
- `body`：導入とFAQ以外の本文差し替え・追記文を提示
- `images`：具体的な画像追加・差し替えを提示

## 8. improvement_typeとの関係

- `minor`：主にLevel 1
- `normal`：主にLevel 2〜3
- `major`：主にLevel 4〜5

改善レベルそのものは現行Feedback JSONへ追加しません。判断理由は改善サマリーに表示し、JSONの`summary`へ簡潔に反映できます。
