# Improvement Level Policy

## 1. 責務分離

記事ランクはSIMS-Blog-Managerが分析して提供します。改善レベルはSIMS-Coreが決定します。

- 記事ランク：Search Consoleと記事DBに基づく現状評価
- 改善レベル：現在の記事、検索意図、改善目的、変更方針を総合した実行判断

記事ランクと改善レベルは同一ではありません。

## 2. 外部から送られた改善レベル

依頼文に改善レベル、リライトレベル、修正強度などが含まれていても、SIMS-Coreの最終判断を拘束しません。

外部指定の改善レベルは採用せず、SIMS-Coreが次の情報から再判定します。

1. 現在の記事本文
2. メインクエリと上位クエリ
3. Search Console概要
4. 記事ランク
5. 改善目的と改善優先順位
6. 変更方針
7. 本文の異常、重複、検索意図とのズレ

## 3. 改善レベル

### Level 1：最小改善

- 誤字、表現、軽微なCTR改善
- 構成と本文をほぼ維持
- S記事を中心に適用

### Level 2：部分改善

- タイトル、説明文、導入、FAQなど限定範囲を改善
- 本文の良い部分を最大限維持
- S・A記事を中心に適用

### Level 3：積極改善

- 複数見出し、本文補足、FAQ、内部リンクを改善
- 検索意図の不足を補う
- A・B記事を中心に適用

### Level 4：構成改善

- H2/H3の追加、削除、名称変更、並べ替え
- 重複や不要部分の整理
- 本文の広範な差し替えを許容
- B・C記事を中心に適用

### Level 5：再構築

- 現在の記事を参考資料として扱う
- 検索意図を満たす構成へ再設計する
- 全面書き換えに近い変更を許容する
- C・D記事や、検索意図との大きな不一致がある記事で検討する

Level 5でも、現在の記事に含まれる独自情報、実体験、正確な手順、有用な説明、既存の良い内部リンクは可能な限り維持します。ゼロから無関係な新規記事を作りません。

## 4. 記事ランクとの関係

記事ランクは重要な判断材料ですが、改善レベルを機械的に決めません。

- Sでも重大な事実誤認があればLevel 3以上になり得る
- Aでも検索意図とのズレが大きければLevel 4になり得る
- Dでも有用な本文が多ければLevel 4に留められる

## 5. 改善サマリーへの表示

改善サマリーには、SIMS-Coreが決定した改善レベルと簡潔な理由を必ず表示します。

例：

```text
改善レベル：Level 2（部分改善）
判断理由：記事ランクAで本文の検索意図一致度が高く、CTR改善を目的にタイトル・導入・FAQへ修正を限定したため。
```

```text
改善レベル：Level 5（再構築）
判断理由：記事ランクDで、本文に別テーマの見出しが混在し、主要検索意図への回答順序も大きくずれているため。
```

## 6. Current Article First

SIMS-Coreは現在の記事を基礎資料として改善します。改善後の記事全文は、現在の記事へ修正内容を反映した完成版です。

ただしLevel 5では、現在の記事を素材として再構築し、使える部分だけを残して新しい構成へ組み直します。
