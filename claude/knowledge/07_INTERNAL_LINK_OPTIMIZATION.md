# Internal Link Optimization

## 1. 目的

SIMS-Blog-Managerから提供される内部リンク候補を評価し、読者導線と記事品質を優先して、採用・保留・不採用を判断します。内部リンクを増やすこと自体を目的にしません。

## 2. 入力

各候補は原則として次の情報を持ちます。

- 記事タイトル
- URL
- 推奨アンカーテキスト
- 関連クエリ
- 関連度

候補は3〜8件程度を想定します。候補が少ない、または0件でも正常に処理します。

## 3. 評価基準

候補ごとに次を確認します。

1. 対象記事の主要検索意図と一致するか
2. 読者が次に知りたい情報か
3. 本文内に自然な設置場所があるか
4. リンク先が対象記事を補完するか
5. 対象記事とリンク先の役割が過度に重複しないか
6. アンカーテキストが不自然にならないか
7. 内部リンク追加で本文が冗長にならないか

関連度の星は参考情報です。星が高くても本文に合わなければ採用しません。

## 4. 判定

- 採用：本文へ自然に追加でき、読者に有益
- 保留：関連はあるが、本文中より関連記事欄やブログカード向き
- 不採用：検索意図、主題、読者導線のいずれかと合わない

採用件数は0件でも構いません。原則1〜3件、長文や網羅記事でも最大5件程度とし、件数合わせで追加しません。

## 5. テキストリンク

テキストリンクを採用する場合は、リンク設定を別欄に分けるだけでは不十分です。変更後の完成文章へ、次のHTMLを直接埋め込みます。

```html
<a href="完全URL">自然なアンカーテキスト</a>
```

出力は必ず次の形にします。

- 変更対象箇所
- 変更前
- 変更後（HTMLリンク埋め込み済みの完成文）
- 修正理由

変更後は、利用者がそのままコピーして既存記事へ差し替えられる状態にします。

### 例

変更前：

```html
<p>NotionのPC版では、Shift＋Enterを使うと同じブロック内で改行できます。</p>
```

変更後：

```html
<p>NotionのPC版では、Shift＋Enterを使うと同じブロック内で改行できます。ブラウザ版との違いやPC版の基本操作も確認したい方は、<a href="https://example.com/notion-desktop">Notionデスクトップアプリ完全ガイド</a>もあわせてご覧ください。</p>
```

## 6. ブログカード

リンク先が独立した詳しい解説記事で、本文中へテキストリンクを入れるより関連記事として明確に案内する方が自然な場合は、ブログカードを選びます。

ブログカードを提案する場合は、必ず次を明記します。

- 挿入対象の見出し
- 前後の段落との位置関係
- リンク先記事タイトル
- 完全URL
- 採用理由

「関連記事として追加」だけでは不十分です。

### 出力例

- 挿入位置：H2「Notionのテーブル内で改行する方法」の本文末、次のH2の直前
- リンク先記事：Notionテーブルの幅変更・揃え方
- ブログカードURL：https://example.com/notion-table-width
- 理由：独立した操作ガイドとして案内する方が本文の流れを壊さないため

## 7. テキストリンクとブログカードの選択

テキストリンクを優先する場面：

- 用語や操作の補足として文脈内で自然に案内できる
- 読者がその場で追加情報へ進む価値が高い
- 一文を追加しても本文の流れが崩れない

ブログカードを優先する場面：

- リンク先が独立した詳しいガイド
- H2を読み終えた後の関連記事として適切
- 文中へ入れると不自然または冗長
- 視覚的に区切って案内する方が理解しやすい

## 8. 既存内部リンク

本文データから既存内部リンクを確認できる場合は、維持・置換・移動・削除を判断できます。

既存リンクを変更する場合も、必ず変更前と変更後を示します。新しいテキストリンクはアフター本文へHTMLで埋め込みます。

既存リンクのURLや位置が確認できない場合、推測で削除や置換を指示しません。

## 9. 内部リンク評価レポート

候補を確認した事実と判断根拠を残すため、内部リンク評価を標準出力します。

採用候補：

- 候補番号
- 判定
- 設置場所
- アンカーテキストまたはブログカード
- アフター完成文またはカード挿入位置
- 理由

保留・不採用候補：

- 候補番号
- 判定
- 簡潔な理由

評価レポートは、コピペ用のアフター本文に代わるものではありません。採用したテキストリンクは必ずアフター本文にも反映します。

## 10. 禁止事項

- 候補を全件採用する
- 関連性が低い候補を件数合わせで追加する
- URLを捏造する
- 「こちら」「詳しくはこちら」だけをアンカーにする
- 完全一致キーワードを不自然に繰り返す
- 設置場所が存在するか確認せず指示する
- URLとアンカーを別欄に示すだけで、アフター本文へ埋め込まない
- ブログカードの位置を曖昧にする

## 11. 将来拡張

将来カニバリ候補が入力された場合は、内部リンク、役割分担、記事統合、別記事維持を判断します。現時点では、内部リンク候補だけを根拠にカニバリ判定を行いません。
