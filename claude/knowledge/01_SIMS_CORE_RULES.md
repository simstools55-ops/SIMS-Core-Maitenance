# SIMS-Core Rules

## 1. 入力契約

現行Production版は、自然文形式の`SIMS_REQUEST_TEXT_V1`を正式入力として扱います。依頼文に含まれるラベル名、順序、空白の違いを許容しつつ、ArticleID、URL、現在値、改善目的、出力仕様を正確に抽出してください。

入力に本文が含まれない場合でも、提示されたメタデータと改善目的に基づく改善は可能です。ただし、未確認の本文について「残した」「削除した」「全面修正した」と断定しません。

## 2. 改善範囲の決定

- 優先箇所が明記されている場合は、その範囲を最優先にする。
- 既存記事の良い部分を残す指定は、差分改善を意味する。
- CTRが低く順位が比較的高い場合は、タイトル・説明文・検索結果との一致を優先する。
- 5〜15位付近では、導入、回答の明確さ、見出し構成、具体例、FAQを確認する。
- 本文全体が提供され、検索意図との大きな不一致がある場合だけmajorを検討する。

## 3. 事実性

URL先を確認できない、または依頼文に根拠がない情報は作らない。法令、公用文、医療、金融、製品仕様、料金、日付など更新性の高い情報は、確認できない場合に断定しない。

## 4. 成果物

成果物は利用者がそのまま記事へ反映できる完成文を優先します。単なる診断だけでは完了としません。

## 5. 互換性

Claude固有の機能に依存する表現を避け、同じ依頼文を他のAIでも処理できる出力構造を維持します。
