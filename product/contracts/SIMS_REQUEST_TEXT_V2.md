# SIMS_REQUEST_TEXT_V2

## 1. 目的

SIMS-Blog-Manager Product 5.2以降からSIMS-Coreへ渡す、記事改善依頼の標準形式です。V1の基本情報に、Search Console上位クエリ、改善優先順位、記事ランク、変更方針、本文JSON、内部リンク候補を追加します。

## 2. 必須項目

- 改善要求文
- ArticleID
- URL
- 記事タイトル
- SEOタイトル
- メタディスクリプション
- メインクエリ
- Search Console概要：クリック、表示回数、CTR、平均順位
- 改善目的
- 改善優先順位
- 記事ランク
- 変更方針
- SIMS_FEEDBACK_V1 version 1.1の出力ルール

## 3. 標準追加項目

### Search Console上位クエリ

最大20件。各項目は次を含みます。

- クエリ
- クリック
- 表示回数
- CTR
- 平均順位

### 記事本文

`SIMS_ARTICLE_SOURCE_V1`形式のJSONを標準とします。

### 内部リンク候補

3〜8件程度。各候補は次を含みます。

- 記事タイトル
- URL
- 推奨アンカーテキスト
- 関連クエリ
- 関連度

### 内部リンク利用ルール

- 候補は参考情報
- すべて使用する必要はない
- 本文の流れと検索意図に合う候補だけ採用
- アンカーテキストは自然に変更可能

## 4. 解釈規則

1. ArticleIDとURLは原文のまま保持する。
2. 数値は推測・補完しない。
3. 記事ランクは現状評価として扱う。改善レベルはSIMS-Coreが現在の記事・検索意図・改善目的・変更方針から決定する。
4. 上位クエリから主要・補助・不足検索意図を判断する。
5. 既存記事の良い部分を維持し、明示されていない全面リライトを行わない。
6. 内部リンク候補は全件評価するが、無理に採用しない。
7. 採用したテキストリンクは、アフター完成文へHTMLリンクとして直接埋め込む。
8. ブログカードが適切な場合は、挿入位置、記事タイトル、完全URL、理由を明記する。
9. 依頼文に改善レベル、リライトレベル、修正強度が含まれていても、SIMS-Coreは採用せず独自に再判定する。
10. 標準出力は、改善サマリー、ビフォー・アフター、内部リンク評価、改善後の記事全文、Feedback JSONの順とする。
11. 出力JSONの仕様は依頼文中の指定を優先するが、記事全文はJSONへ含めない。

## 5. 後方互換性

SIMS-CoreはV1依頼も引き続き処理します。V2項目がない場合は、入力された情報の範囲で改善し、不足項目を推測で補完しません。
