# User Guide

## 日常運用

1. SIMS-Blog-Managerで改善対象を選ぶ。
2. 改善依頼文をコピーする。
3. ClaudeのSIMS-Core Projectへ貼り付ける。
4. 「変更後」の完成文を記事へコピーして反映する。テキストリンクはHTML埋め込み済みです。ブログカードは指定位置へ完全URLを貼り付けます。
5. 回答末尾のFeedback JSONをBlog-Managerへ登録する。
6. 指定日数後に効果を確認する。

## 注意

記事本文がClaudeから確認できない場合は、本文または必要な見出し・該当箇所を追加してください。本文を確認できていない出力を、そのまま全面反映しないでください。


## 内部リンクの反映

- テキストリンク：アフター文章に`<a href="URL">アンカーテキスト</a>`が埋め込まれているため、そのまま差し替えます。
- ブログカード：指定された見出し・段落位置へ、記載された完全URLをブログカードとして貼り付けます。
- 不採用候補：記事品質を優先した判断のため、追加する必要はありません。


## 標準出力（R4）

SIMS-Coreは次の順で出力します。

1. 改善サマリー
2. ビフォー・アフター
3. 内部リンク評価
4. 改善後の記事全文（コピペ用）
5. SIMSフィードバックJSON

記事ランクはSIMS-Blog-Managerから受け取りますが、改善レベルはSIMS-Coreが本文と検索意図を確認して決定します。依頼文に改善レベルが含まれていても、その指定には従いません。

改善後の記事全文は利用者向けです。SIMSフィードバックJSONには全文を含めません。


## R5出力品質ルール

- Before／Afterのコードブロック内には、コピー対象以外のラベルを入れません。
- 改善後の記事全文は導入文から文末の内部リンク・ブログカードまで含みます。
- ブログカードは完全URLを完成記事の指定位置へ反映します。
