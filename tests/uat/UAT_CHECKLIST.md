# UAT Checklist

## 入力処理

- [ ] ArticleIDを正しく認識した
- [ ] URLを正しく認識した
- [ ] メインクエリを正しく認識した
- [ ] GSC数値を取り違えていない
- [ ] 改善目的と優先箇所を守った
- [ ] Search Console上位クエリを正しく解釈した
- [ ] 記事ランクを現状評価として利用した
- [ ] 改善レベルをCore側で独自決定した
- [ ] 依頼文中の改善レベル指定に拘束されていない
- [ ] 改善レベルと判断理由を改善サマリーへ表示した

## 改善品質

- [ ] 一般論ではなく完成文がある
- [ ] タイトル案が検索意図と本文に一致する
- [ ] 導入文で結論が早く示される
- [ ] 見出しの回答順序が明確
- [ ] FAQが本文の単純重複ではない
- [ ] 未確認情報を捏造していない
- [ ] 変更箇所がビフォー・アフター＋理由で示されている
- [ ] BeforeとAfterが独立したコードブロックで表示された
- [ ] H1・SEOタイトル・説明文がtext形式で表示された
- [ ] アフター文章がそのままコピペ可能
- [ ] 改善後の記事全文が出力された
- [ ] 全文へすべての修正と内部リンクが反映された

## 内部リンク

- [ ] 内部リンク候補を全件確認した
- [ ] 採用・保留・不採用の判断が妥当
- [ ] 採用テキストリンクがアフター本文へHTMLで埋め込まれている
- [ ] HTMLに完全URLと自然なアンカーテキストがある
- [ ] 採用テキストリンクのMarkdown版も出力された
- [ ] ブログカードの挿入見出しと前後位置が明確
- [ ] ブログカードの記事タイトルと完全URLがある
- [ ] 関連性の低い候補を件数合わせで採用していない
- [ ] 既存リンクの変更を推測で指示していない

## Feedback JSON

- [ ] JSONは1つだけ
- [ ] JSONは回答末尾
- [ ] ArticleIDとURLが正しい
- [ ] changesが出力内容と一致
- [ ] new_valuesが提示値と一致
- [ ] 列挙値が契約内
- [ ] recommended_review_daysが7・14・30
- [ ] JSONの後に文章がない
- [ ] JSONに改善後の記事全文が含まれていない

## 判定

- [ ] PASS
- [ ] CONDITIONAL PASS
- [ ] FAIL
